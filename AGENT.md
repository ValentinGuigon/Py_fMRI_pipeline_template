# Agent Reference: SLB fMRI Pipeline

GLM-based fMRI analysis pipeline for the SLB dataset. Runs subject- and group-level analyses from fMRIPrep outputs through to thresholded stat maps, PDF reports, and per-task-group analysis indexes. Built around FitLins inside a patched Apptainer container.

---

## Repo root = WORK_ROOT

The repo root on the server is `/data/sld/homes/vguigon/slb_work`. All relative paths in scripts default to this location. The variable `WORK_ROOT` in `run_pipeline.sh` and `submit_pipeline.sh` defaults to this value; override only if the repo is cloned elsewhere.

## Task groups

| Group | BIDS tasks | Data dictionary | Events type | Notes |
|-------|-----------|-----------------|-------------|-------|
| `tmth` | `tm`, `th` | Trust-DataDictionaryfMRI.csv | standard + motor | Trust/money hypothetical task |
| `ol` | `obslearn` | OL-DataDictionaryfMRI.csv | standard + motor | Observational learning |
| `sra` | `riskself` | SRA-DataDictionaryfMRI.csv | standard + motor | Self risk assessment |

**Standard events**: use events TSVs already in `slb_bids_runs/` (via `slb_events/original/`).
**Motor events**: use `slb_events/motor/` which adds three delta trial types per button press: `button_press`, `button_press_left`, `button_press_right`. Set `EVENTS_DIR` in the config to activate.

### scripts/run/ вҖ” pipeline execution

| Script | Purpose | Key invocation |
|--------|---------|----------------|
| `run_pipeline.sh` | **Main entry point.** Orchestrates: fitlins вҶ’ fixreport вҶ’ plot-run вҶ’ plot-group вҶ’ report. Loads a config file; all parameters auto-derived from model JSON if not set. | `bash run_pipeline.sh --config fitlins_configs/{group}/{stem}.cfg --steps all` |
| `submit_pipeline.sh` | SLURM wrapper for `run_pipeline.sh`. Generates a timestamped sbatch script and submits it. All `run_pipeline.sh` flags pass through. | `bash submit_pipeline.sh --config fitlins_configs/{group}/{stem}.cfg --steps all` |
| `run_fitlins_models.sh` | Low-level FitLins runner inside the Apptainer container. Called by `run_pipeline.sh`; rarely invoked directly. | `bash run_fitlins_models.sh --bids-dir {dir} --model {json} --subjects "000 001"` |
| `parse_model.py` | Extract `subjects`, `tasks`, `contrasts`, `stat`, `run_node`, `group_nodes` from a model JSON. Used by `run_pipeline.sh` for auto-derivation; can be called directly for inspection. | `python3 parse_model.py {model}_smdl.json --field subjects` |
| `fix_fitlins_reports.py` | Patch FitLins HTML reports: rewrite broken container-internal image paths, embed all figures as base64, inject missing contrast figures. Modifies the HTML in place; writes a `.bak` backup. | `python3 fix_fitlins_reports.py {out_dir}/reports/model-*.html [--verbose]` |
| `plot_fmri_statmaps.py` | Nilearn-based stat map plotter. Produces glass brain, slice, optional 3D HTML, and optional TSV cluster table. Writes a `manifest.tsv`. Called by `run_pipeline.sh`; can be invoked directly. | `python3 plot_fmri_statmaps.py --root {out_dir} --nodes node-{name} --glob "**/*stat-t*_statmap.nii*" --outdir {fig_dir} --thr-mode p-unc --p-unc 0.01 --two-sided` |

## Key constraints

- **Always use the patched container.** The vanilla container will fail on this dataset.
- **Standard visual models** set `EVENTS_DIR` to `slb_events/original/` in their config. **Motor models** set it to `slb_events/motor/`. The `events_type` field in the analysis index JSON encodes this.
- **Never edit `slb_events/original/`**. It is a verbatim reference copy. Only `slb_events/motor/` is modified.
- **`slb_bids_runs/` NIfTIs are symlinks** into the shared source. Deleting or overwriting them affects the shared tree. Metadata (TSV, JSON) are real copies and safe to edit.
- **FitLins will not re-run if the output directory already exists**, unless `--force` is passed. Check `fitlins_derivatives/{group}/{stem}_s{kernel}/` before re-running.
- **`work_fitlins/` is ephemeral** and can be large. It is safe to delete after a successful run.
- **Adding a new model**: create `{stem}_smdl.json` in `fitlins_models/{task_group}/` and `{stem}.cfg` in `fitlins_configs/{task_group}/`, following existing files in the same group as templates. Validate the model JSON with `validate_bids_stats_model.py` before running.
